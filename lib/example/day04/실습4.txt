Day110 [SPRING JPA + Flutter] 실습4 : TODO 등록, 전체조회
    - Spring 서버에서 JPA로 데이터를 관리하고, Flutter 앱에서 Dio로 데이터를 주고받습니다.
    - flutter : day03 , day04 예제 참고

[조건1] 엔티티 설계
    - Todo : 기본적인 할 일 정보를 저장 , id(pk/auto_increment) , title, content
    - 데이터베이스 테이블명은 todo 로 지정
    - BaseTime : 생성일(createdAt), 수정일(updatedAt)을 자동 저장
    - toDto() 포함

[조건2] DTO / Repository / Service / Controller 설계
    - DTO는 Entity ↔ DTO 변환을 위한 toEntity( ) 포함
    - Repository는 JpaRepository를 상속
    - 등록 (createTodo), 전체조회 (getTodoList)

[조건3] Flutter 연동, Dio 패키지로 Spring 서버와 통신
    - POST : 새로운 TODO 등록
    - GET : 전체 TODO 목록 조회

[조건4] 하단 바텀 탭 구성
    - 홈 페이지 탭 화면(선택)
    - 등록 페이지 탭 화면
    - 전체 조회 페이지 탭 화면
